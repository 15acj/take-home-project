# Plan: Pull top 10k most-cited works from OpenAlex

## Context
We need a reproducible script that fetches the 10,000 most-cited scholarly works from the
OpenAlex API into local files for downstream analysis. The user wants to (1) run a small
validation batch first to inspect the returned fields and confirm correctness, then (2) do
the full 10k run. The user also asked whether we need to batch/parallelize and about rate
limits. This plan answers those from **live API validation done during planning** (not memory),
since OpenAlex has recently changed its docs and introduced a metered cost model.

Working dir `/Users/andrewjin/Desktop/anthropic-take-home` has empty `scripts/` and `web/`
dirs. Python 3.12.5 is present; `requests`/`httpx`/`pandas` are **not** installed.

## Decisions (confirmed with user)
- **Scope:** All work types (no `type:` filter) — truly the top-10k cited works of any kind.
- **Anomalies:** Keep the raw top-10k, but add a heuristic `suspect` flag column; drop nothing.
- **Output:** Raw JSONL (full fidelity) **+** a flattened CSV for eyeballing.

## What was validated live (api.openalex.org)
- Endpoint/sort works: `GET /works?sort=cited_by_count:desc`. `meta.count` ≈ 217M works.
- **Cursor paging works** (`cursor=*` → `meta.next_cursor`); `per_page` max = 200.
- **Cost model:** every response now carries `meta.cost_usd`. It was a flat **$0.0001/request**
  whether `per_page=2` or `200`. Docs mention a **$1/day complimentary** allowance
  (≈10,000 requests/day free). Our full job is ~50 requests ≈ **$0.005** — negligible.
- **No API key was required** for these calls (only `mailto` was sent). Current docs say a free
  key is "required"; empirically `mailto` alone still works. We'll support an optional key.
- **Data has real anomalies at the very top.** The #1 result — "Radiation Resistant Camera
  System…" (2020) — reports 801,217 citations but **801,210 of them land in a single year
  (2025)**, `fwci`=0.975 (dead average), not even top-10%. Ranks 2–10 (R, Lowry, Laemmli,
  Bradford, ResNet, PBE/GGA…) are legitimately among the most-cited works ever.
- **A naive heuristic was tested and rejected:** `sum(counts_by_year)/cited_by_count` does NOT
  separate anomalies — `counts_by_year` only covers ~2012+, so genuine pre-2012 classics score
  low (0.17) while the anomaly scores 1.00. Do not use it. Use single-year concentration instead.

## Deliverables
- `scripts/fetch_top_cited.py` — the script (stdlib `urllib` only, so no install step needed;

- `scripts/requirements.txt` — empty/none needed for the fetcher (note this in README).
- Outputs (written to `scripts/data/` or a `--out-dir`):
  - `top_cited.jsonl` — one raw OpenAlex work object per line.
  - `top_cited.csv` — flattened, inspectable columns.

- Endpoint/sort works: `GET /works?sort=cited_by_count:desc`. `meta.count` ≈ 217M works.

## Script design
Single script, parameterized so the sample run and full run use identical code:

```

```

politeness delay between requests).

cursor is null. `per_page=min(200, remaining)`. ~50 requests for 10k. Retry with exponential

3. **Impact mismatch:** `cited_by_count > 100000` but `fwci` is not null and `fwci < 5`

that in the README and keep every record.

**Flattened CSV columns:**

citation_normalized_percentile_value, first_author, num_authors, source_name, oa_status,

## Batching / parallelization / rate limits (the user's explicit questions)
- **No parallelization needed.** 10,000 ÷ 200 = ~50 requests. Cursor paging is inherently
  sequential (each request needs the previous `next_cursor`). At a ~0.1s polite delay that's a
  few seconds total. Parallelizing would require basic `page` paging (page 1–50, independent),
  which *is* possible because basic paging is capped at exactly 10,000 results — but it adds
  complexity and 429 risk for no meaningful gain. Recommend **sequential cursor paging**.
- **Rate limits:** historically OpenAlex allowed ~10 req/s and 100k/day; the new metered model
  meters by `cost_usd` (~$0.0001/req, $1/day ≈ 10k req free). Our ~50-request job is far under
  any limit and costs ≈ $0.005. We still add a small delay + `mailto` (polite pool) + backoff.
  (Note: the live rate-limits doc page 404'd/moved during planning; numbers above combine the
  historical figures with the cost model observed live — worth a final confirm, but immaterial
  at this volume.)

#!/usr/bin/env python3
"""Fetch the top-N most-cited works from the OpenAlex API.

Pulls works sorted by citation count (descending) using cursor paging, writes the
raw records to JSONL and a flattened summary to CSV, and adds a heuristic `suspect`
flag to help spot known OpenAlex citation-count anomalies (kept, never dropped).

Stdlib only - no pip install required.

Examples:
    # small validation batch
    python scripts/fetch_top_cited.py --limit 15 --out-dir scripts/data/sample

    # full run
    python scripts/fetch_top_cited.py --limit 10000 --out-dir scripts/data/full
"""

from __future__ import annotations

import argparse
import csv
import json
import os
import sys
import time
import urllib.error
import urllib.parse
import urllib.request
from datetime import datetime, timezone

API_URL = "https://api.openalex.org/works"

# Fields requested from OpenAlex. Kept lean but rich enough to inspect and to compute
# the suspect flag. abstract_inverted_index / referenced_works lists are omitted (large,
# not needed for a citation ranking) - easy to add here later if wanted.
SELECT_FIELDS = [
    "id",
    "doi",
    "ids",
    "title",
    "display_name",
    "publication_year",
    "publication_date",
    "type",
    "language",
    "cited_by_count",
    "counts_by_year",
    "fwci",
    "citation_normalized_percentile",
    "authorships",
    "primary_location",
    "locations_count",
    "open_access",
    "biblio",
    "referenced_works_count",
    "primary_topic",
    "is_retracted",
    "is_paratext",
]

CSV_COLUMNS = [
    "rank",
    "openalex_id",
    "doi",
    "title",
    "type",
    "publication_year",
    "cited_by_count",
    "fwci",
    "citation_normalized_percentile_value",
    "first_author",
    "num_authors",
    "source_name",
    "oa_status",
    "language",
    "is_retracted",
    "is_paratext",
    "suspect",
    "suspect_reasons",
]

MAX_PER_PAGE = 200


def build_url(cursor: str, per_page: int, mailto: str, api_key: str | None) -> str:
    params = {
        "sort": "cited_by_count:desc",
        "per_page": str(per_page),
        "cursor": cursor,
        "select": ",".join(SELECT_FIELDS),
        "mailto": mailto,
    }
    if api_key:
        params["api_key"] = api_key
    return f"{API_URL}?{urllib.parse.urlencode(params)}"


def fetch_page(url: str, max_retries: int = 5) -> dict:
    """GET one page, retrying with exponential backoff on 429/5xx."""
    attempt = 0
    while True:
        attempt += 1
        req = urllib.request.Request(url, headers={"User-Agent": "top-cited-fetcher/1.0"})
        try:
            with urllib.request.urlopen(req, timeout=60) as resp:
                return json.loads(resp.read().decode("utf-8"))
        except urllib.error.HTTPError as e:
            # 429 (rate limit) and 5xx are transient -> back off and retry.
            if e.code == 429 or 500 <= e.code < 600:
                if attempt > max_retries:
                    raise RuntimeError(
                        f"Giving up after {max_retries} retries (last HTTP {e.code})."
                    ) from e
                wait = min(2 ** attempt, 60)
                print(f"  HTTP {e.code}; backing off {wait}s (attempt {attempt})", file=sys.stderr)
                time.sleep(wait)
                continue
            # Other 4xx are fatal (bad query, etc.) - surface the body to help debug.
            body = e.read().decode("utf-8", "replace")[:500]
            raise RuntimeError(f"HTTP {e.code} fetching page: {body}") from e
        except (urllib.error.URLError, TimeoutError) as e:
            if attempt > max_retries:
                raise RuntimeError(f"Network error after {max_retries} retries: {e}") from e
            wait = min(2 ** attempt, 60)
            print(f"  network error {e}; retrying in {wait}s (attempt {attempt})", file=sys.stderr)
            time.sleep(wait)


def compute_suspect(work: dict) -> tuple[bool, str]:
    """Heuristic flag for likely citation-count anomalies. Human-inspection aid only.

    See plan: single-year citation concentration is the robust signal; sum(counts_by_year)
    vs total is NOT (counts_by_year only covers ~2012+).
    """
    reasons: list[str] = []
    total = work.get("cited_by_count") or 0

    # 1. Single-year spike: most citations landing in one year is the smoking gun
    #    for merge/count artifacts (e.g. 801,210 of 801,217 in a single year).
    counts = work.get("counts_by_year") or []
    if total > 1000 and counts:
        max_year = max((c.get("cited_by_count") or 0) for c in counts)
        if max_year / total > 0.90:
            reasons.append(f"single_year_spike({max_year}/{total})")

    # 2. Retracted / paratext.
    if work.get("is_retracted"):
        reasons.append("retracted")
    if work.get("is_paratext"):
        reasons.append("paratext")

    # 3. Impact mismatch: six-figure citations but field-weighted impact ~average.
    fwci = work.get("fwci")
    if total > 100000 and fwci is not None and fwci < 5:
        reasons.append(f"low_fwci({fwci})")

    return (bool(reasons), ";".join(reasons))


def flatten(work: dict, rank: int, suspect: bool, reasons: str) -> dict:
    authorships = work.get("authorships") or []
    first_author = None
    if authorships:
        author = authorships[0].get("author") or {}
        first_author = author.get("display_name")

    source_name = None
    primary_location = work.get("primary_location") or {}
    source = primary_location.get("source") or {}
    source_name = source.get("display_name")

    oa = work.get("open_access") or {}
    cnp = work.get("citation_normalized_percentile") or {}

    return {
        "rank": rank,
        "openalex_id": work.get("id"),
        "doi": work.get("doi"),
        "title": work.get("display_name") or work.get("title"),
        "type": work.get("type"),
        "publication_year": work.get("publication_year"),
        "cited_by_count": work.get("cited_by_count"),
        "fwci": work.get("fwci"),
        "citation_normalized_percentile_value": cnp.get("value"),
        "first_author": first_author,
        "num_authors": len(authorships),
        "source_name": source_name,
        "oa_status": oa.get("oa_status"),
        "language": work.get("language"),
        "is_retracted": work.get("is_retracted"),
        "is_paratext": work.get("is_paratext"),
        "suspect": suspect,
        "suspect_reasons": reasons,
    }


def main() -> int:
    p = argparse.ArgumentParser(description="Fetch top-N most-cited works from OpenAlex.")
    p.add_argument("--limit", type=int, default=10000, help="Number of works to pull (default 10000).")
    p.add_argument("--per-page", type=int, default=MAX_PER_PAGE, help="Results per request (max 200).")
    p.add_argument("--out-dir", default="scripts/data/full", help="Output directory.")
    p.add_argument("--mailto", default="15andrewj@gmail.com", help="Polite-pool contact email.")
    p.add_argument("--api-key", default=None, help="Optional OpenAlex API key (not required).")
    p.add_argument("--sleep", type=float, default=0.1, help="Delay between requests, seconds.")
    args = p.parse_args()

    per_page = max(1, min(args.per_page, MAX_PER_PAGE))
    os.makedirs(args.out_dir, exist_ok=True)
    jsonl_path = os.path.join(args.out_dir, "top_cited.jsonl")
    csv_path = os.path.join(args.out_dir, "top_cited.csv")
    meta_path = os.path.join(args.out_dir, "run_meta.json")

    cursor = "*"
    fetched = 0
    total_cost = 0.0
    n_requests = 0
    n_suspect = 0
    api_total_count = None
    started = time.time()

    print(f"Fetching top {args.limit} works -> {args.out_dir}", file=sys.stderr)

    with open(jsonl_path, "w", encoding="utf-8") as jf, open(csv_path, "w", newline="", encoding="utf-8") as cf:
        writer = csv.DictWriter(cf, fieldnames=CSV_COLUMNS)
        writer.writeheader()

        while fetched < args.limit and cursor:
            page_size = min(per_page, args.limit - fetched)
            url = build_url(cursor, page_size, args.mailto, args.api_key)
            data = fetch_page(url)
            n_requests += 1

            meta = data.get("meta") or {}
            total_cost += meta.get("cost_usd") or 0.0
            api_total_count = meta.get("count", api_total_count)
            cursor = meta.get("next_cursor")

            results = data.get("results") or []
            if not results:
                break

            for work in results:
                if fetched >= args.limit:
                    break
                fetched += 1
                suspect, reasons = compute_suspect(work)
                if suspect:
                    n_suspect += 1
                jf.write(json.dumps(work, ensure_ascii=False) + "\n")
                writer.writerow(flatten(work, fetched, suspect, reasons))

            print(
                f"  req {n_requests}: +{len(results)} (total {fetched}/{args.limit}), "
                f"cost so far ${total_cost:.4f}",
                file=sys.stderr,
            )
            if fetched < args.limit and cursor:
                time.sleep(args.sleep)

    elapsed = time.time() - started
    run_meta = {
        "fetched_at": datetime.now(timezone.utc).isoformat(),
        "limit": args.limit,
        "records_written": fetched,
        "requests": n_requests,
        "per_page": per_page,
        "total_cost_usd": round(total_cost, 6),
        "suspect_count": n_suspect,
        "api_reported_total_works": api_total_count,
        "elapsed_seconds": round(elapsed, 1),
        "endpoint": API_URL,
        "sort": "cited_by_count:desc",
        "select_fields": SELECT_FIELDS,
        "mailto": args.mailto,
        "used_api_key": bool(args.api_key),
    }
    with open(meta_path, "w", encoding="utf-8") as mf:
        json.dump(run_meta, mf, indent=2)

    print(
        f"\nDone: {fetched} records in {n_requests} requests, {elapsed:.1f}s, "
        f"cost ${total_cost:.4f}, {n_suspect} flagged suspect.\n"
        f"  {jsonl_path}\n  {csv_path}\n  {meta_path}",
        file=sys.stderr,
    )
    return 0


if __name__ == "__main__":
    sys.exit(main())

Available but NOT currently pulled

OpenAlex works also expose these — any can be added to SELECT_FIELDS:

  • abstract_inverted_index — the abstract (word→positions map; reconstructable to text). Omitted: large.
  • referenced_works — full list of cited work IDs (not just the count). Omitted: large — up to hundreds of IDs each.
  • related_works — OpenAlex-computed similar works.
  • topics / concepts / keywords / sifted — the full topic list (vs just primary_topic), legacy concepts, keywords.
  • locations / best_oa_location — all hosting locations (vs just primary + count).
  • grants — funder + award IDs.
  • sustainable_development_goals — UN SDG tags.
  • mesh — MeSH terms (biomedical).
  • apc_list / apc_paid — article processing charges.
  • indexed_in, countries_distinct_count, institutions_distinct_count, corresponding_author_ids, cited_by_api_url, updated_date, created_date, fulltext_origin, has_fulltext, is_authors_truncated.
