Cause

Because the set height and scrollHeight can differ by a sub-pixel from line-height rounding, the browser rendered a vertical scrollbar even on a single line — the "scrollbar for no reason."

  • onChatInput switches overflowY to "auto" only when content actually exceeds the 96px cap (i.e., a genuinely long, multi-line message that needs to scroll), and back to "hidden" otherwise.
  • After sending, the input's height reset also resets overflowY to "hidden".

Net effect: no scrollbar while typing normal one- or two-line messages; a scrollbar appears only once the input grows past 96px and truly needs one.
